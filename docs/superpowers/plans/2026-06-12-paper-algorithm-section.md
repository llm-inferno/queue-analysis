```python
def fig5_onset_search():
    """Method schematic: closed-form markers + anchor + downward search to M*.
    Illustrative (baseline data); the measured evaluation lives in Experiments."""
    truth = load_truth("baseline")
    p = SCN_PARAMS["baseline"]
    m_star = truth["M_truth"]
    f_star = truth["throughput_truth"]
    ms = [pt["m"] for pt in truth["f_curve"]]
    fs = [pt["throughput"] for pt in truth["f_curve"]]
    f_of = {pt["m"]: pt["throughput"] for pt in truth["f_curve"]}

    EPS = 0.02
    m_itl = P.m_itl(p, M_MAX)
    m_tpf = P.m_tpf(p, M_MAX)
    # Bracket exactly as formula_guided.py computes it (itl-or-crossover cell).
    seed = min(max(m_itl, m_tpf), M_MAX)
    lo = max(1, min(m_itl, m_tpf))
    hi = min(seed, 3 * m_itl, M_MAX)
    f_anchor = f_of[M_MAX]
    threshold = (1.0 - EPS / 2.0) * f_anchor

    # Replay the downward binary search to mark the probed midpoints.
    a, b, probes = lo, hi, []
    iters = 0
    while a < b and iters < 6:
        mid = (a + b) // 2
        probes.append(mid)
        if f_of[mid] >= threshold:
            b = mid
        else:
            a = mid + 1
        iters += 1

    fig, ax = plt.subplots(figsize=(5.8, 3.4))
    ax.plot(ms, fs, color="C0", zorder=2)
    # Search bracket the algorithm narrows.
    ax.axvspan(lo, hi, alpha=0.08, color="C0", zorder=0)
    # Anchor and threshold (horizontal references).
    ax.axhline(f_anchor, color="C3", linestyle="-.", linewidth=1.0, zorder=1)
    ax.axhline(threshold, color="C3", linestyle=":", linewidth=1.0, zorder=1)
    ax.text(M_MAX * 0.62, f_anchor * 1.01, r"anchor $f^*=f(M_{\max})$",
            color="C3", fontsize=8, va="bottom")
    ax.text(M_MAX * 0.62, threshold * 0.985, r"threshold $(1-\varepsilon/2)f^*$",
            color="C3", fontsize=8, va="top")
    # Closed-form markers and onset.
    for x, c, lab in [(m_itl, "C2", rf"$M_{{\mathrm{{ITL}}}}={m_itl}$"),
                      (m_star, "gray", rf"$M^*={m_star}$"),
                      (m_tpf, "C1", rf"$M_{{\mathrm{{TPF}}}}={m_tpf}$")]:
        ax.axvline(x, color=c, linestyle="--", linewidth=1.2, zorder=1)
    # Downward-search probes: vertical ticks at each probed M on the curve.
    for mid in probes:
        ax.plot([mid], [f_of[mid]], marker="v", color="C4", markersize=6, zorder=3)
    ax.annotate("downward search", xy=(probes[-1], f_of[probes[-1]]),
                xytext=(lo + 6, f_star * 0.45), color="C4", fontsize=8,
                arrowprops=dict(arrowstyle="->", color="C4", lw=0.8))
    handles = [
        plt.Line2D([0], [0], color="C2", linestyle="--", label=rf"$M_{{\mathrm{{ITL}}}}={m_itl}$ (lower bd)"),
        plt.Line2D([0], [0], color="C1", linestyle="--", label=rf"$M_{{\mathrm{{TPF}}}}={m_tpf}$ (TTFT upper bd)"),
        plt.Line2D([0], [0], color="gray", linestyle="--", label=rf"onset $M^*={m_star}$"),
        plt.Line2D([0], [0], color="C4", marker="v", linestyle="", label="search probes"),
    ]
    ax.legend(handles=handles, loc="lower right", framealpha=0.9, fontsize=8)
    ax.set_xlabel("MaxBatchSize $M$")
    ax.set_ylabel("$f(M)$ [RPS]")
    ax.set_xlim(0, M_MAX)
    ax.set_ylim(0, f_star * 1.15)
    fig.savefig(FIG_DIR / "fig5_onset_search.pdf")
    plt.close(fig)
```

```latex
\section{Algorithm: Formula-Guided Onset Search}
\label{sec:algorithm}

The analysis of \S\ref{sec:analysis} reduces optimisation to \emph{onset detection}:
because $f(M)$ rises to a plateau (\textbf{P1}), the operating point of interest is the
smallest $M$ that reaches near-peak throughput,
\begin{equation}
  \hat M \;=\; \min\{\, M : f(M) \ge (1-\varepsilon)\,f^\ast \,\},
  \qquad \varepsilon = 0.02,
  \label{eq:objective}
\end{equation}
where the peak $f^\ast$ is not known in advance and must itself be probed. Two naive
strategies bracket the design space. Returning $M_{\max}$ costs a single oracle call but
over-provisions concurrency, defeating the purpose of optimising the onset. A
parameter-blind search that simply \emph{maximises} $f$ converges to a point on the
plateau interior --- the wrong objective for Eq.~\eqref{eq:objective} --- and spends many
oracle calls doing so. We instead exploit the closed-form structure of
\S\ref{sec:analysis}: warm-start from the closed-form primitives (\textbf{P2}), then run a
short downward monotone-predicate search (\textbf{P3}) whose threshold is anchored to avoid
the occupancy-gap trap (\textbf{P4}). Algorithm~\ref{alg:onset} states the procedure.

\begin{algorithm}[t]
  \DontPrintSemicolon
  \KwIn{params; oracle $f(\cdot)$; bounds $m_{\min}, m_{\max}$; tolerance $\varepsilon$}
  \KwOut{onset estimate $\hat M \in [m_{\min}, m_{\max}]$}
  $M_{\mathrm{ITL}} \leftarrow$ largest $B$ with $\mathrm{itl}(B) \le \mathrm{targetITL}$
    \tcp*{closed form, no oracle calls}
  $M_{\mathrm{TPF}} \leftarrow$ largest $B$ with $\mathrm{ttft\_prefill}(B) \le \mathrm{targetTTFT}$\;
  \eIf{$M_{\mathrm{TPF}} < M_{\mathrm{ITL}}$}{
    $\mathit{seed} \leftarrow m_{\max}$ \tcp*{ttft-only cell: $M_{\mathrm{TPF}}$ undershoots}
  }{
    $\mathit{seed} \leftarrow \min(\max(M_{\mathrm{ITL}}, M_{\mathrm{TPF}}), m_{\max})$\;
  }
  $f^\ast \leftarrow f(\mathit{seed})$ \tcp*{1 oracle call}
  \If{$f^\ast \le 0$ \textbf{and} $\mathit{seed} < m_{\max}$}{
    $\mathit{seed} \leftarrow m_{\max}$;\quad $f^\ast \leftarrow f(\mathit{seed})$\;
  }
  \lIf{$f^\ast \le 0$}{\Return $m_{\max}$ \tcp*[f]{infeasible everywhere probed}}
  $\theta \leftarrow (1 - \varepsilon/2)\, f^\ast$ \tcp*{margin keeps a confirmatory call $<\varepsilon$}
  $\mathit{lo} \leftarrow \max(m_{\min},\, \min(M_{\mathrm{ITL}}, M_{\mathrm{TPF}}))$\;
  \eIf{$M_{\mathrm{TPF}} < M_{\mathrm{ITL}}$}{
    $\mathit{hi} \leftarrow \min(\mathit{seed}, m_{\max})$\;
  }{
    $\mathit{hi} \leftarrow \min(\mathit{seed},\, 3 M_{\mathrm{ITL}},\, m_{\max})$
      \tcp*{occupancy-gap bound}
  }
  $\mathit{hi} \leftarrow \max(\mathit{lo}, \mathit{hi})$;\quad $i \leftarrow 0$\;
  \While{$\mathit{lo} < \mathit{hi}$ \textbf{and} $i < \textsc{MaxIters}$}{
    $\mathit{mid} \leftarrow \lfloor (\mathit{lo}+\mathit{hi})/2 \rfloor$\;
    \eIf{$f(\mathit{mid}) \ge \theta$}{$\mathit{hi} \leftarrow \mathit{mid}$}{$\mathit{lo} \leftarrow \mathit{mid}+1$}
    $i \leftarrow i + 1$\;
  }
  \Return $\mathrm{clamp}(\mathit{hi}, m_{\min}, m_{\max})$\;
  \caption{\textsc{FormulaGuidedOnsetSearch} ($\textsc{MaxIters}=6$)}
  \label{alg:onset}
\end{algorithm}
```

```latex
\subsection{Why each step}
\label{sec:algo-rationale}

\paragraph{Closed-form seed (P2).}
The primitives $M_{\mathrm{ITL}}$ and $M_{\mathrm{TPF}}$ are computed from the service
model alone, at zero oracle cost (\S\ref{sec:mitl}, \S\ref{sec:regimes}). The seed is
\emph{cell-aware}: on the TTFT-only cell ($M_{\mathrm{TPF}} < M_{\mathrm{ITL}}$),
$M_{\mathrm{TPF}}$ \emph{undershoots} the onset because the realised TTFT frontier sits
above the prefill-only bound once M/M/1 queue wait is included, so there we seed the
$f^\ast$ probe at $m_{\max}$ instead of at the constraint endpoint.

\paragraph{High anchor at $f(M_{\max})$ (P4).}
This is the load-bearing choice. Seeding the near-peak reference $f^\ast$ at the
constraint endpoint $U=\max(M_{\mathrm{ITL}},M_{\mathrm{TPF}})$ can badly
\emph{underestimate} the peak: when queue wait lets realised throughput climb far above
the per-iteration binding batch size, $f(U) \ll f^\ast$, the threshold $\theta$ is set too
low, and the descent halts prematurely. On one benchmark scenario the occupancy gap
reaches $6.3\times$ ($U = 9$ while the plateau peak sits at $M \approx 57$); a
$U$-anchored threshold stops the search at $M = 9$, a $22\%$ throughput shortfall that
violates the $\varepsilon = 2\%$ tolerance. Anchoring $f^\ast$ at $f(M_{\max})$ reads the
true plateau height and eliminates the failure, at the cost of one seed probe.

\paragraph{Bracket.}
The lower end $\mathit{lo} = \min(M_{\mathrm{ITL}}, M_{\mathrm{TPF}})$ floors the search at
$M_{\mathrm{TPF}}$ on the TTFT-only cell, where $M_{\mathrm{ITL}}$ overshoots the onset.
The upper end is tightened by the occupancy-gap bound $3M_{\mathrm{ITL}}$ (the gap stays
within this factor on the scenarios studied), relaxed to the seed on the TTFT-only cell
where $M_{\mathrm{ITL}}$ is an unreliable tightener.

\paragraph{Threshold margin.}
Searching to $\theta = (1-\varepsilon/2)f^\ast$ rather than $(1-\varepsilon)f^\ast$ leaves
headroom so that a downstream confirmatory evaluation of the returned $\hat M$ still lands
within the $\varepsilon$ SLO-feasibility tolerance.

Figure~\ref{fig:onset-search} illustrates the procedure on the baseline scenario:
the closed-form markers, the anchored threshold, and the downward probes converging to
$M^\ast = 69$ from the bracket $[40, 120]$.

\begin{figure}[t]
  \centering
  \includegraphics[width=0.85\linewidth]{fig5_onset_search}
  \caption{Formula-guided onset search on the baseline. Closed-form $M_{\mathrm{ITL}}$ and
           $M_{\mathrm{TPF}}$ bracket the search; $f^\ast$ is anchored at $f(M_{\max})$ and
           the threshold set at $(1-\varepsilon/2)f^\ast$; the downward monotone-predicate
           probes converge to the onset $M^\ast$. Illustrative; the measured evaluation is
           in \S\ref{sec:experiments}.}
  \label{fig:onset-search}
\end{figure}

\subsection{Guarantees}
\label{sec:algo-guarantees}

\paragraph{Cost.}
The procedure issues at most one seed probe, $\textsc{MaxIters}=6$ search probes, and one
downstream confirmatory call --- eight oracle calls in total --- and is $O(\log M_{\max})$
in the search range (\textbf{P3}).

\paragraph{Correctness.}
Because $f$ is non-decreasing to a plateau (\textbf{P3}), the downward monotone-predicate
search returns the smallest $M$ in the bracket with $f(M) \ge \theta$, so the result meets
the near-peak objective of Eq.~\eqref{eq:objective}. The single high anchor is safe while
the post-peak decline of $f$ stays below $\varepsilon/2$ (\S\ref{sec:search}); under
chunk-count jumps that produce a larger decline, a multi-probe $f^\ast$ would be required
(left to future work).

\paragraph{Onset, not argmax.}
The search deliberately targets the onset $M^\ast$, not the strict throughput argmax, which
can sit far higher on a plateau that wiggles at floating-point precision
(\S\ref{sec:search}). Over-provisioning to the argmax wastes concurrency; any residual gap
between the returned onset and the strict argmax is a structural property of the plateau,
not a defect of the search.

\subsection{Baselines}
\label{sec:algo-baselines}

We compare against two strategies that span the cost/quality extremes. \emph{Naive-max}
returns $M_{\max}$ in a single oracle call but over-provisions concurrency.
\emph{Parameter-blind ternary search} maximises $f$ over $[m_{\min}, m_{\max}]$; on a
monotone-to-plateau curve it converges to an interior plateau point rather than the onset,
solving the wrong objective, and spends roughly thirty oracle calls doing so. The
formula-guided search Pareto-dominates both on the concurrency objective; the measured
comparison across the benchmark is reported in \S\ref{sec:experiments}.

\subsection{Provenance}
\label{sec:algo-provenance}

The closed-form primitives, the three-cell regime partition, and the high-anchor fix were
not hand-designed but \emph{discovered} by an automated formula-discovery campaign that
iteratively proposed and oracle-validated candidate structure against the queueing model.
The campaign's recorded principles (RP-1 through RP-11) are the evidence base for the
design choices above: the exact closed form under $nc=1$ (RP-1), the occupancy-gap
anchor fix (RP-2), the monotone-to-plateau shape (RP-3), the regime partition and its
load-bearing ordering signal (RP-6, RP-7), and the call-budget result (RP-8).
```
